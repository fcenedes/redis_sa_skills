# Selectors and Locators

How to pick locators that survive UI refactors.

## Priority Order

1. **Role + accessible name** — `page.getByRole('button', { name: 'Sign in' })`. Survives style and DOM changes; aligns with how users and assistive tech identify elements.
2. **Label** — `page.getByLabel('Email')`. Stable for forms.
3. **Placeholder** — `page.getByPlaceholder('you@example.com')`. Use when no label.
4. **Text** — `page.getByText('Welcome back', { exact: true })`. Good for headings, banners, links.
5. **Alt text / title** — `page.getByAltText('Redis logo')`, `page.getByTitle('Settings')`.
6. **Test ID** — `page.getByTestId('submit')`. Use only when no semantic locator works.

Never reach for CSS or XPath unless every option above fails.

## Good Locators

```ts
page.getByRole('heading', { name: 'Cluster overview' });
page.getByRole('link', { name: /docs/i });
page.getByLabel('Database name');
page.getByPlaceholder('redis-cloud-12345');
page.getByText(/^Connected$/);
page.getByTestId('shard-row-3');
```

Chain locators to scope:

```ts
const row = page.getByRole('row', { name: /shard 3/i });
await row.getByRole('button', { name: 'Restart' }).click();
```

## Bad Locators

```ts
page.locator('div.card > div:nth-child(2) span');     // brittle CSS
page.locator('//div[@class="card"]//span[2]');         // XPath
page.locator('.btn.btn-primary');                       // class soup
page.locator('text=Sign in >> nth=0');                  // ordering-dependent
page.locator('[id^=mui-]');                             // generated IDs
```

## When to Add `data-testid`

Add a test id only when:

- The element has no role, label, placeholder, or unique text.
- Multiple identical elements exist and ordering is unstable.
- The visible text is i18n'd and changes per locale.
- A page object needs a stable handle for a non-semantic container.

Name test ids by **intent**, not implementation: `submit-order`, not `red-button`.

## Accessibility Implications

`getByRole` and `getByLabel` exercise the same affordances assistive tech uses. If a test cannot find an element by role or label, your app likely has an a11y gap — fix the app, then update the test. Don't paper over a11y bugs with `data-testid`.

## Locator Anti-Patterns

- Locators that depend on visual position (`nth=0`, `:first-child`).
- Locators that include translated strings without a fallback.
- Locators tied to CSS classnames generated by build tooling (Tailwind hashes, CSS modules, Emotion).
- Locators that traverse component internals (`>> internal:role=button`).
- Locators chained more than two levels deep — usually a sign the DOM needs a stable hook.
